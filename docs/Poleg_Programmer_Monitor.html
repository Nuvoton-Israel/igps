1.  Features:

·       SPI programming

·       Fuse Arrays programming

·       Fuse Arrays read (into memory area)

·       Initialize DDR (needed for SPI programming).

2.  Work concept:

·       Run from SRAM2 0xFFFD6000, since it will initialize DDR (if not initialized yet) and other required modules.

·       Check Command TAG at 0xFFFDC000.

·       Execute the command according to the TAG.

·       Echo trace/debug info into SRAM2 0xFFFDBF00 and above

·       Return with the result in ARM CPU r0 register (0 = success, other = error)

3.  Supported Commands (written to address 0xFFFDC000):

·       Memory Controller initialization

Ø O: offset 0x00 size 4: tag: Memory Controller initialization TAG (0xA50750A9)

·       SPI programming

Ø O: offset 0x00 size 4: tag: SPI programming TAG (0xA5075001)

Ø O: offset 0x04 size 4: size: Size to program

Ø O: offset 0x08 size 4: address: Absolute address (code will check if offset + size is inside SPI

Ø O: offset 0x0C size 4: fiu_dwr_cfg_set: Word contents copied to relevant FIU FIU_DWR_CFG register - Ignored if either 0 or 0xFFFFFFFF

Ø O: offset 0x10 size 4: fiu_drd_cfg_set: Word contents copied to relevant FIU FIU_DRD_CFG register - Ignored if either 0 or 0xFFFFFFFF

Ø O: offset 0x14 size 1: fiu_clk_divider: Defines the clock divide ratio from AHB to FIU0 clock - Ignored if either 0 or 0xFF

Ø O: offset 0x15 size 1: erase_cmd: Command for erasing the smallest sector, if 0 the command will be 0x20 - sector erase

Ø O: offset 0x16 size 1: erase_size_power: The size of the erased area in power of 2: ie. if erase_size_power = 12, than the size is 2^12=4096

Ø O: offset 0x17 size 1: Reserved

Ø O: offset 0x18 size 4: crc: CRC of binary (Not in use yet)

Ø O: offset 0x1C size 4: binary_address: The address of the binary to copy from into the SPI

·       Fuse programming

Ø O: offset 0x00 size 4: tag: Fuse programming TAG (0xA5075002)

Ø O: offset 0x04 size 4: size: Size to program (max 1024)

Ø O: offset 0x08 size 4: offset: Offset in fuse array, code will check if offset + size is not bigger than fuse size (1024 bytes)

Ø O: offset 0x0C size 1: fuse_array_num: Fuse array number: 0  Key array, 1  Fuse array

Ø O: offset 0x0D size 1: check_parity: Check if FUSE programming will not cause nibble parity error or majority error (Not in use yet)

Ø O: offset 0x0E size 10: Reserved

Ø O: offset 0x18 size 4: crc: CRC of binary (Not in use yet)

Ø O: offset 0x1C size 4: binary_address: The address of the binary to copy from into the fuses

·       Fuse read

Ø O: offset 0x00 size 4: tag: Fuse read TAG (0xA5075003)

Ø O: offset 0x04 size 8: size: Size to read (max 1024)

Ø O: offset 0x08 size 4: offset: Offset in fuse array, code will check if offset + size is not bigger than fuse size (1024 bytes)

Ø O: offset 0x0C size 1: fuse_array_num: Fuse array number: 0  Key array, 1  Fuse array

Ø O: offset 0x0D size 15: Reserved

Ø I: offset 0x1C size 4: binary_address: The address to copy into from the fuses

4.  Log Trace (at SRAM2 0xFFFDBF00):

·       offset 0x00 size 4: log_last_status: The latest logged status (if bit 31 is set the following log_last_data has valid value)

·       offset 0x04 size 4: log_last_data: The latest logged data (relevant if bit 31 of log_last_status is set)

·       offset 0x08 size 4: log_next_address: Points to the next free log address

·       offset 0x0C size 4: reserved

·       offset 0x10 size 240: log_buffer: contains the trace of the log status and optionally log data one after another:

o   log_status: The logged status (if bit 31 is set the following log_data has valid value)

o   log_data: The logged data (exist only if bit 31 of previous log_status is set)

6.  Operation flow (can be done by UART Update tool or by JTAG):

·       Initializing DDR

1.    Load ARM Programmer Monitor binary to 0xFFFD6000

2.    Copy Memory Controller initialization TAG (0xA50750A9) to address 0xFFFDC000

3.    Run the ARM Programmer Monitor code at 0xFFFD6000 and wait until it is return - this will initialize the DDR

4.    Check its return value at ARM CPU r0 register (0 = success, other = error)

·       Copy to SPI (without reset after initializing DDR)

1.    Prepare SPI programming header binary with binary_address (0x8000 for example)

2.    Load the SPI programming header binary to address 0xFFFDC000

3.    Prepare the desired binary image (i.e. header_body.bin)

4.    Copy header_body.bin to binary_address (0x8000 for example)

5.    Run the ARM Programmer Monitor code at 0xFFFD6000 and wait until it is return - this will copy the header_body.bin to SPI

6.    Check its return value at ARM CPU r0 register (0 = success, other = error)

·       Copy to Fuses (In this example it does not use DDR since the image is up to 4096 bytes)

1.    Load ARM Programmer Monitor binary to 0xFFFD6000

2.    Prepare Fuse programming header binary with binary_address (0xFFFE0000 for example)

3.    Load the Fuse programming header binary to address 0xFFFDC000

4.    Prepare the desired binary fuse image (i.e. fuse.bin)

5.    Copy fuse.bin to binary_address (0xFFFE0000 for example)

6.    Run the ARM Programmer Monitor code at 0xFFFD6000 and wait until it is return - this will copy the fuse.bin to the fuses

7.    Check its return value at ARM CPU r0 register (0 = success, other = error)

7.  Future enhancements:

·       Check if CRC field matches the binary data to program if not exit with error

·       Check if FUSE programming will not cause nibble parity error or majority error
